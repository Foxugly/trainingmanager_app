# S1d-b — i18n (5 locales) + LanguageService Implementation Plan

> Executed inline (autonomous). Completes S1 (apart from Turnstile-blocked register/forgot-password + resend).

**Goal:** Localize the whole UI in fr/nl/en/it/es with a live in-app language switch, persisted to `me.language` and reflected in the `Accept-Language` header.

**Architecture (mirrors PushIT's `Strings`/`LocalStrings` pattern):**
- `i18n/Strings.kt` — a flat `Strings` interface (one property per message, deduplicated across screens) + `languageDisplayNames` + `supportedLanguages` + `stringsFor(lang)` + `LocalStrings` (`staticCompositionLocalOf`, default `StringsFr`).
- `i18n/StringsFr|Nl|En|It|Es.kt` — one `object` per locale implementing `Strings` (fr is the source; nl/en/it/es translated).
- `i18n/LanguageService.kt` — Koin **single**: `activeLang` (Compose snapshot, default fr), `strings` (computed), `setActive(lang)` (local; also sets `LanguageProvider.activeTag` for Accept-Language), `suspend switchLanguage(lang)` (optimistic + `PATCH /me/`, rollback on failure).
- **UI** reads `LocalStrings.current`; **ViewModels** receive the active `Strings` at construction (`get<LanguageService>().strings` from the Koin factory; default `StringsFr` so tests need no wiring) for error messages.
- `App()` wraps content in `CompositionLocalProvider(LocalStrings provides languageService.strings)` (recomposes on switch) and, at bootstrap when authenticated, seeds the language from `GET /me/`.
- The Profile language selector calls `languageService.switchLanguage(code)`; language is removed from `ProfileViewModel.save` (handled solely by `LanguageService`).

## Tasks (executed)
1. **i18n infra** — `Strings` interface + 5 locale objects + `LanguageService` (+ `LanguageServiceTest`: setActive updates tag/strings; switch success persists; switch failure rolls back).
2. **Migrate logic** — delete the 5 per-screen FR `*Strings` objects; `mapLoginError(throwable, strings)`; every text-producing ViewModel gains a `strings: Strings = StringsFr` param (Login, MagicLinkRequest, ResetPassword, Invitation, ChangePassword, Profile); `ProfileViewModel` drops `LanguageProvider` + language-in-save.
3. **Migrate UI** — every screen reads `val s = LocalStrings.current`; Home inline strings localized; Profile language dropdown wired to `LanguageService`; `AppModule` adds `single LanguageService` and passes `get<LanguageService>().strings` to VM factories; `App` provides `LocalStrings` + bootstrap language seed.
4. **Adapt tests** — assertions reference `StringsFr.*`; `mapLoginError` called with `StringsFr`; `ProfileViewModelTest` rewritten (no LanguageProvider / no language field).

**Verify:** `:composeApp:testAndroidHostTest` green; `:androidApp:assembleDebug` builds. iOS/Android entry points unchanged (App's new params default via `koinInject()`).
